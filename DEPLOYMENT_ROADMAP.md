# Portfolio release roadmap

이 문서는 공개 배포 전후에 확인할 항목을 우선순위별로 관리한다.

## P0 — 공개 전 필수

- [x] 화면에서 전화번호와 이력서 원본을 제외한다.
- [x] `RESUME*.pdf`, 환경파일, 개발 서버 로그를 Git에서 제외한다.
- [x] 사업장 코드 대신 공개 가능한 집계값인 `8 manufacturing sites`만 사용한다.
- [x] 내부 고유 용어를 일반적인 애플리케이션·서비스 식별자 표현으로 바꾼다.
- [x] 현재 파일에서 비밀키, 토큰, 인증서, 데이터베이스 파일이 없는지 검색한다.
- [x] Firewatch에서 가져온 이미지 자산을 현재 소스에서 제거한다.
- [ ] 회사명, 고객명, 화면 캡처, 정량 성과를 공개해도 되는지 회사 정책과 NDA 기준으로 최종 확인한다.
- [ ] 공개 연락처로 현재 Gmail 주소를 계속 사용할지 결정한다.
- [ ] MES 프로젝트 저장소의 정확한 GitHub URL과 공개 범위를 확인한다.
- [ ] 과거 Git 기록에 남은 Firewatch 자산을 정리하거나 깨끗한 공개 저장소를 새로 만든다.
- [ ] 최종 한국어·영어 문구를 본인이 사실관계 기준으로 검수한다.

## P1 — 첫 공개 품질

- [x] 한국어/영어 전환과 브라우저 언어 기본값을 지원한다.
- [x] 본문, 사례 설명, 태그, 모바일 글자 크기를 상향한다.
- [x] 단일 페이지에서 불필요한 React Router를 제거한다.
- [x] Netlify 빌드·리다이렉트·기본 보안 헤더를 설정한다.
- [ ] 데스크톱과 모바일에서 줄바꿈, 메뉴, 애니메이션, 링크를 직접 확인한다.
- [ ] MES 프로젝트마다 기밀 데이터를 제거한 합성 데이터 기반 화면 또는 짧은 데모를 만든다.
- [ ] `문제 → 내가 한 일 → 결과` 수치의 산정 기준을 한 문장씩 보강한다.
- [ ] 대표 이미지와 Open Graph 메타데이터를 추가해 링크 공유 미리보기를 만든다.
- [ ] CRA 개발 도구 의존성 경고를 줄이기 위해 Vite로 이전한다.

## P2 — 공개 후 개선

- [ ] 개인정보를 수집하지 않는 방문 통계를 선택적으로 연결한다.
- [ ] MES 진단 프레임워크 아키텍처 다이어그램을 추가한다.
- [ ] CIM–MES Diff Check의 익명화된 입력/출력 예시를 추가한다.
- [ ] 연구 프로젝트를 MES 경력과 분리된 `Independent Research`로 유지하고 논문 링크를 보강한다.
- [ ] 필요할 때 개인 도메인을 연결한다. 도메인 없이도 Netlify 기본 주소로 공개 가능하다.

## 배포 순서

1. `npm ci`와 `npm run build`가 성공하는지 확인한다.
2. P0의 미완료 항목을 본인이 승인한다.
3. Netlify에서 저장소를 연결하고 프로젝트 루트를 이 폴더로 지정한다.
4. 배포 미리보기 주소에서 한국어/영어, 모바일, 외부 링크를 확인한다.
5. 기본 `*.netlify.app` 주소를 공개한다.
